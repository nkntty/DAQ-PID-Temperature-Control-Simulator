# DAQ-PID-Temperature-Control-Simulator

A simple simulation of a thermal system controlled using a PID controller.
Demonstrates feedback control, data acquisition concepts, and numerical simulation.


Quickstart

g++ PIDcontroller.cpp -o pid_sim -std=c++17

./pid_sim

python3 plot_results.py

Output
Explain what columns mean:
time (step or seconds)
temp
setpoint
control (heater power 0–1)


1. 制御システムの重要性 (Control Systems Relevance)
制御システムは、入力された目標値に対して出力値を望ましい状態に調整・維持するための仕組みです。現代社会において不可欠な技術であり、以下の理由から極めて重要です。 
自動化と効率化: 人間の介在を減らし、24時間体制での精密な動作や高速な処理を実現します（例：工場のロボットアーム、洗濯機の自動サイクル）。
安全性と安定性の確保: 航空機のオートパイロットや自動車のABS（アンチロック・ブレーキ・システム）のように、極限状態でもシステムを安定させ、事故を未然に防ぎます。
精度の向上: 半導体製造や医療機器（人工呼吸器など）において、微細な誤差も許されない環境での正確な動作を保証します。

2. DAQシミュレーションの概念 (DAQ Simulation Concept)
DAQ (Data Acquisition：データ収集) とは、温度や圧力などの物理現象を電気信号に変え、コンピュータで扱えるデジタルデータに変換するプロセスです。 
DAQシミュレーションは、物理的なハードウェア（センサーやボード）が手元にない状態でも、ソフトウェア上で仮想のDAQデバイスを動作させる手法です。 
目的: 実際の機器を繋ぐ前に、プログラムの構文チェックやロジックの検証を行うために使用されます。
メリット: 高価な機材を壊すリスクを避け、開発コストを抑えながら、想定されるデータ入力に対するシステムの反応をテストできます。

3. C++が組み込み・制御システムで使われる理由
C++は、ハードウェアに近い低レイヤの制御と、複雑なソフトウェア設計を両立できるため、制御分野の標準言語となっています。 
ハードウェアへの直接アクセス: C言語と同様に、メモリのアドレスやハードウェアレジスタを直接操作できるため、センサーやモータの精密な制御が可能です。
ゼロオーバーヘッド抽象化: 「使わない機能にコストを払わない」という設計思想があり、オブジェクト指向などの高度な機能を使っても、実行速度を犠牲にしません。
決定論的な実行速度: ガベージコレクション（自動メモリ回収）による予測不能な停止が発生しないため、ミリ秒単位の応答が求められるリアルタイム制御に適しています。
保守性と再利用性: クラスや継承（OOP）を用いることで、複雑な制御ロジックをモジュール化し、大規模なプロジェクトでもコードの再利用や管理が容易になります。 
MATLAB/Simulink などのツールで設計した制御モデルを、最終的に高速な C++コード として出力して実機に実装する流れが一般的です。
